# 67 Instruction 67
Геймифицированная платформа лояльности для хакатона.  
Пользователи выполняют квесты, получают бонусы, открывают достижения, покупают награды и крутят колесо фортуны.  
Вся активность сопровождается уведомлениями, которые отправляются во внешний Telegram-сервис.

Инструкция по нашему проекту для хакатона.
Директории:
- Frontend (работа приложения, но с ней вы можете ознакомиться на отдельной страничке гитхаба)
- Backend (серверная часть (Боллее детально ее структура представлена в `backend\README.md`))
- Telegram (связка с ТГ-ботом согласно заданию хакатона)

## Архитектура проекта и стек

- Проект создан на ЯП **Python 3.13**
- **requirements.txt** - библиотеки, которые используется с ним.
- БД в рамках хакатона это либо **PostgreSQL** или **SQLite** (для того, чтобы быстро проверить)
- **FastAPI** — веб-фреймворк

- [Backend](backend/README.md)

### Слои проекта
#### 1. Контроллеры (`routers/`)
Этот слой отвечает только за маршрутизацию, валидацию входящих JSON-данных (через схемы Pydantic) и проверку прав доступа (JWT-токены).
То есть здесь находятся ЭНДПОИНТЫ проекта
#### 2. Слой бизнес-логики (`services/`)
Слой содержит изолированные функции, которые реализуют игровые правила и транзакции.
Здесь находится система наград, интеграция с внешними системами и т.д.
#### 3. Слой данных / Модели (`tabels/`)
Вместе с `database.py` представляют собой базу данных проекта.
Декларативные модели SQLAlchemy, описывающие таблицы, типы полей (Integer, String, DateTime) и связи между ними (Foreign Keys, Relationships).

## Бизнес-логика

- **Ежедневный вход** — начисляется `DAILY_BONUS_AMOUNT` бонусов, увеличивается счётчик серии дней. При возвращении после перерыва серия сбрасывается, отправляется событие «реактивация».
- **Квесты** — один пользователь может выполнить квест только один раз. За выполнение начисляется указанное количество бонусов, обновляется лидерборд текущего месяца.
- **Магазин наград** — списание бонусов, уменьшение остатка товара.
- **Колесо фортуны** — раз в день бесплатно, последующие вращения платные. Выбор приза по заданным вероятностям, может дать бонусы или предмет из магазина.
- **Достижения** — выдаются автоматически при выполнении условий (первый квест, 5/10 квестов, серии дней 3/7/30). Каждое достижение можно получить только один раз.

| **Variable** | **Default** | **Description** |
| ------------- | ------------- | ------------- |

| `ACCESS_TOKEN_EXPIRE_MINUTES` | `60` | `Время жизни сессионного токена авторизации пользователя (в минутах) после успешного логина.`|

# Быстрый запуск проекта
1. Подготовка базы данных
Убедитесь, что на вашем компьютере запущен локальный сервер PostgreSQL. С помощью pgAdmin, DBeaver или консоли psql создайте пустую базу данных:
```
CREATE DATABASE sks_quest;
```

2. Настройка окружения
В корневой папке создайте файл `.env` и укажите ваши локальные доступы к СУБД:
```
DATABASE_URL=postgresql://postgres:ТВОЙ_ПАРОЛЬ_ОТ_POSTGRES@localhost:5432/sks_quest
SECRET_KEY=ЛюбаяСлучайнаяДлиннаяСтрокаДляЗащитыТокенов2026
```

3. Установка зависимостей и запуск
Активируйте ваше виртуальное окружение и запустите сервер автоматической перезагрузки Uvicorn:
```
# Активация окружения (Windows PowerShell)
.\\.venv\\Scripts\\Activate.ps1

# Установка библиотек
pip install -r requirements.txt

# Переход в папку бэкэнда если это нужно
cs .\backend

# Запуск бэкенда
uvicorn rest:app --reload
```
